## Overview

ScaryMoovies uses a four-scale color system built around a single design philosophy: **dark-first, horror-glamorous, and high-contrast**. Every UI surface starts from near-black and earns light through hierarchy, not convention.

The four scales each carry a distinct role:

- **Purple** is the brand's soul — a deep, saturated violet that reads as horror-glamour rather than corporate. It is reserved exclusively for interactive elements: buttons, links, focus rings, and active states.
- **Red** is used sparingly, like blood. It signals danger, error, and horror accents — the genre marker on a "Coming Soon" badge or the pulse behind an achievement alert. Overuse would dilute its impact; it should always feel like something is wrong, in the best possible way.
- **Black** is the void — the ground everything else sits on. The scale runs from near-total darkness (black-900) up to a dark mid-gray (black-50). Three stops define the surface system: the primary background, the card surface, and the elevated/raised layer.
- **White** is used exclusively for typography and light decorative elements. It never appears as a background. Its scale tapers from near-pure white (white-50, the primary text color) down to a dim gray (white-900) for disabled and placeholder states.

Together these four scales produce an interface that feels cinematic: saturated accents floating over a lightless void, with text that reads at maximum clarity.

## Accessibility

All contrast ratios are calculated against the primary background (`--bg-primary`, `#0c0a0e`). WCAG 2.1 thresholds: **AA requires 4.5:1** for normal text, **3:1** for large text (18pt+ or 14pt bold); **AAA requires 7:1** for normal text.

| Foreground | Background | Approx. Contrast | WCAG Rating |
|---|---|---|---|
| white-50 `#fefefe` | black-500 `#0c0a0e` | ~20.8:1 | AAA |
| white-700 `#adacae` | black-500 `#0c0a0e` | ~10.0:1 | AAA |
| white-900 `#666667` | black-500 `#0c0a0e` | ~4.6:1 | AA |
| purple-300 `#785492` | black-500 `#0c0a0e` | ~4.8:1 | AA (large text / UI components) |
| purple-400 `#5d337d` | black-500 `#0c0a0e` | ~3.1:1 | AA for UI components only (not body text) |
| red-500 `#751111` | black-500 `#0c0a0e` | ~3.3:1 | AA for UI components only (not body text) |

**Key callouts:**

- `white-50` on `black-500` clears AAA by a wide margin — use it freely for all body copy and headings.
- `white-900` on `black-500` barely clears AA — it is acceptable only for placeholder and disabled states, never for readable content.
- `purple-300` and `purple-400` do not meet AA for body text at normal size. They satisfy AA only as large interactive UI components (buttons, badges) where the 3:1 threshold applies. Never use either as inline text color against dark backgrounds.
- `red-500` follows the same rule as purple: acceptable for large UI components and icon+label pairs, never for body or label text.

## Usage Rules

**Purple is for interaction only.** Do not use any purple-scale token for decorative illustration, text, or backgrounds outside of interactive affordances. Every purple element should respond to a user gesture (hover, focus, active, selected).

**Red is for danger and genre accents only.** It appears on error states, destructive confirmation dialogs, "Coming Soon" or "Sold Out" badges, and horror-genre markers. Do not use red for general decoration or to add visual variety.

**Never use purple-100 or lighter (purple-50, purple-100, purple-200) as text on dark backgrounds.** These steps do not meet contrast requirements and will read as washed out rather than branded.

**Black-600 sits darker than black-500.** This is intentional: surfaces (cards) recede slightly behind the page background, creating depth without borders. Do not invert this relationship.

**White scale is text-only.** No white-scale token should be used as a background color. For light surfaces, use the black scale; for light-mode support (if introduced), a separate light-mode token set will be defined.

**Semantic tokens are the production contract.** Components must reference `--bg-primary`, `--accent-primary`, etc. — not raw scale tokens like `purple-400`. This ensures a future theme swap or dark/light mode toggle can be applied without touching component code.
