# Components

This reference documents every UI component in the ScaryMoovies system. Each entry covers purpose, anatomy, all interactive states with exact color tokens, dimensional specs, and usage rules. Treat these specs as the source of truth — never override with one-off values.

## Navbar

The Navbar is present on every page of the product. It provides brand identity, primary navigation, and the main action controls — including the PYF Button and user account access.

<NavbarShowcase />

### Anatomy

1. **Root bar** — full-width, `height: 64px`, `background: var(--bg-surface)`, `sticky top-0 z-50`
2. **Logo** — left-aligned wordmark, links to `/`
3. **Nav links** — centered; `EXPLORE` and `COLLECTIONS`, all-caps Inter
4. **Action group** — right-aligned; Search icon · PYF Button · Bell icon · Avatar

### Nav Link States

| State   | Color       | Token             |
|---------|-------------|-------------------|
| Default | `white-700` | `--text-secondary` |
| Hover   | `white-50`  | `--text-primary`   |
| Active  | `white-50`  | `--text-primary`   |

### Specs

| Property       | Value                                              |
|----------------|----------------------------------------------------|
| Height         | `64px`                                             |
| Background     | `var(--bg-surface)` (`#0b090d`)                   |
| Position       | `sticky`, `top: 0`, `z-index: 50`                 |
| Nav link size  | `14px`, weight `500`, `0.06em` tracking, all-caps |
| Nav link gap   | `32px`                                             |
| Action gap     | `16px`                                             |
| Icon size      | `20px`, `strokeWidth: 1.5`                        |
| PYF button     | Pill shape, `8px 16px` padding, 75% scale of hero variant |
| Avatar size    | `32px`, `border-radius: 9999px`                   |

### Usage Rules

**Use on:** every page without exception. The Navbar is never hidden, replaced, or removed from the DOM — even on full-screen overlays, it renders beneath.

**Do not:** reduce height below 64px on desktop; use `position: fixed` instead of `sticky`; duplicate the PYF button anywhere else in the Navbar.

<UserAvatarShowcase />

The `ring` prop applies a `2px solid var(--accent-primary)` border. Use it when the avatar represents the **signed-in user** — primarily on the profile page header and the Navbar avatar. Never apply the ring to another user's avatar in a social context.

When no image `src` is provided, renders the user's initials (up to 2 characters) in `--text-secondary` on `--bg-elevated`. The fallback font is Inter, weight 600.

## Three-State Action

The three-state action group lets users log their relationship to a film: **Watched**, **Watchlist**, and **Liked**. Each state is independent and toggleable.

<ThreeStateShowcase />

### States

| Button    | Icon   | Active color       | Meaning                              |
|-----------|--------|--------------------|--------------------------------------|
| Watched   | Eye    | `--accent-hover`   | I have seen this film                |
| Watchlist | Clock  | `--accent-hover`   | I intend to watch this               |
| Liked     | Heart  | `--accent-danger`  | This is one of my favourites         |

### Behavior

- Each button toggles independently. A film can be Watched + Liked simultaneously.
- Active state shows a filled border and background tint using `color-mix` against the accent color.
- Watched and Watchlist use `--accent-hover` (purple) — both are neutral positive actions.
- Liked uses `--accent-danger` (red) — the one moment red functions as passion, not danger.

### Usage Rules

**Place on:** movie detail pages, movie cards on hover, and the profile watchlist row.

**Do not:** replace the three-state group with a single "Add to list" button — users need the distinction between queued and watched. Do not apply the Liked red to other heart/favourite interactions elsewhere in the product — it breaks the red scarcity rule except in this one intentional exception.

## Star Rating & Tier Selector

Two distinct rating input components. Both are used to capture a user's rating for a film — choose the right one based on context.

<StarRatingShowcase />

### Star Rating

Half-star precision (0.0–5.0). Used in the **review submission flow** and displayed as a read-only badge in **review cards** and **search results**.

| Property       | Value                                      |
|----------------|--------------------------------------------|
| Range          | 0.0 – 5.0, half-star increments            |
| Star color     | `--accent-hover` (filled), `--bg-elevated` (empty) |
| Sizes          | `sm` (14px), `md` (20px), `lg` (28px)      |
| Interaction    | Left half of star = 0.5, right half = full |

### Tier Selector

S–F single-select. Used on the **Tier List page** when a user places a film into a tier. The selected tier renders at full opacity; unselected tiers dim to 40%.

**When to use which:**
- **Star rating** → reviews, public scores, comparison contexts
- **Tier selector** → the Tier List feature exclusively

Do not present both rating systems simultaneously for the same film. Pick one per context and be consistent.

## Skeleton Loader

Replaces content during data fetches. Dimensions match the element being loaded — no layout shift when real content arrives.

<SkeletonShowcase />

### Variants

| Component        | What it replaces              |
|------------------|-------------------------------|
| `SkeletonPoster` | Movie poster (2:3 ratio)      |
| `SkeletonCard`   | Full movie card (poster + meta)|
| `SkeletonTextBlock` | Prose text lines           |
| `Skeleton`       | Any single element — pass `width` + `height` |

### Specs

| Property      | Value                                              |
|---------------|----------------------------------------------------|
| Base color    | `--bg-elevated` (`#3d3b3e`)                       |
| Shimmer       | `rgba(255,255,255,0.04)` gradient, left → right   |
| Duration      | `1.5s`, `infinite`                                 |
| Border radius | Matches the element it replaces                    |
| Motion rule   | Shimmer disabled under `prefers-reduced-motion`    |

### Usage Rules

**Always use:** skeleton loaders when a grid or list is waiting on async data. Never show an empty container — it looks like a bug.

**Match dimensions exactly.** A skeleton that is the wrong size causes layout shift when real content arrives.

## Callout

Inline spec callout for documentation use. Three variants: `info`, `warning`, `tip`.

<CalloutShowcase />

### Variants

| Variant   | Icon          | Border              | Use for                                      |
|-----------|---------------|---------------------|----------------------------------------------|
| `info`    | Info circle   | `--border-default`  | General notes, requirements, reminders       |
| `warning` | Triangle      | `--accent-danger`   | Breaking constraints, size limits, avoid-X  |
| `tip`     | Lightbulb     | `--accent-hover`    | Best-practice hints, recommended patterns   |

### Usage

```mdx
<CalloutBox variant="warning">
  Do not use below 20px — legibility breaks at smaller sizes.
</CalloutBox>
```

Use `info` as the default. Only reach for `warning` when the guidance prevents a real mistake. `tip` is for advice that's useful but not blocking.
