# Components

This reference documents the core UI components of the ScaryMoovies interface. Each entry covers purpose, anatomy, all interactive states with exact color tokens, dimensional specs, and rules governing correct use. Treat these specs as the source of truth; never override them with one-off values.

## Button System

The ScaryMoovies button system is built on shadcn/ui with brand token overrides. All interactive states, focus rings, and disabled styles are wired to CSS variables — never hardcoded hex values.

<ButtonShowcase />

### Variants

| Variant | Token | When to use |
|---|---|---|
| `default` | `--accent-primary` bg | Primary action — main CTA, confirm, save |
| `destructive` | `--accent-danger` bg | Irreversible actions — delete, remove, clear data |
| `outline` | `--border-default` border | Secondary action alongside a primary button |
| `secondary` | `--bg-elevated` bg | Supplementary actions — add to list, share |
| `ghost` | transparent, hover fills | Low-emphasis inline actions — skip, mark watched |
| `ghost-danger` | transparent, hover danger | Low-emphasis destructive — clear list, unfriend |
| `link` | underline text | Inline prose actions — read more, see reviews |

### Sizes

| Size | Height | Use case |
|---|---|---|
| `sm` | 36px | Compact UI — table actions, chip controls |
| `default` | 40px | Standard UI — forms, modals, cards |
| `lg` | 44px | Section-level CTAs — collection headers |
| `xl` | 52px | Hero CTAs — landing page, empty states |
| `icon` | 40×40px | Icon-only actions — share, bookmark |

### Focus and Accessibility

All buttons carry a `2px solid var(--ring)` focus outline (`--purple-300`) that appears on keyboard navigation. Never suppress this with `outline: none` without a replacement. Disabled states use `opacity: 50%` and `cursor: not-allowed` — the disabled prop is always required, never achieved through CSS alone.

## Pick Your Fear (PYF) Button

The PYF Button is a distinct component from the button system above. It uses a **pill shape** (`border-radius: 9999px`) and is reserved exclusively for triggering the fear-selection flow.

    <PYFButton />

  </div>

### States

| State    | Background       | Text           | Token                          | Other                          |
|----------|------------------|----------------|--------------------------------|--------------------------------|
| Default  | `--accent-primary` | `--text-primary` | `var(--accent-primary)`      | Resting state                  |
| Hover    | `--accent-hover`  | `--text-primary` | `var(--accent-hover)`        | Lightens on cursor enter       |
| Active   | pressed           | —              | `scale(0.97)` transform        | Momentary press feedback       |
| Disabled | `--black-300`    | `--text-muted` | `var(--black-300)`             | `cursor-not-allowed`, no hover |

### Specs

| Property       | Value                          |
|----------------|--------------------------------|
| Border radius  | `9999px` (pill — never square) |
| Padding        | `12px 24px`                    |
| Font size      | `14px`                         |
| Font weight    | `500` (medium)                 |
| Letter spacing | `0.08em`                       |
| Label          | `PICK YOUR FEAR` — never reworded |
| Transition     | `background 150ms ease`        |
| Min width      | `160px`                        |

### Usage Rules

**Use when:** triggering the fear-selection flow is the intended action and the context supports a high-emphasis CTA (Navbar, hero, empty-state prompts).

**Do not use when:** a secondary or destructive action is needed. Do not duplicate more than twice on a single viewport. Do not relabel — `PICK YOUR FEAR` is brand-critical. Do not use the `Button` component for this — it uses `border-radius: 6px`. PYFButton is its own component.

**Common mistakes:** using `purple-500` as the resting background (that is the Active token); applying rounded-md instead of pill radius; nesting inside another interactive element.

## Movie Poster — Vertical

### Purpose

The vertical poster represents a single film in portrait orientation. It is the most common content unit across the product — appearing in collection grids, watchlist layouts, and as thumbnails inside Tier List rows.

### Anatomy

1. **Image frame** — 2:3 aspect ratio container, `border-radius: 8px`
2. **Poster image** — fills frame via `object-fit: cover`
3. **Hover overlay** — semi-transparent scrim (`black/50`) that fades in on hover
4. **Rating chip** — surfaces on hover, positioned bottom-right of the overlay

### States

| State  | Transform        | Overlay opacity | Rating chip |
|--------|------------------|-----------------|-------------|
| Rest   | `scale(1.00)`    | `0`             | Hidden      |
| Hover  | `scale(1.03)`    | `1` (black/50)  | Visible     |

### Specs

| Property         | Value                                |
|------------------|--------------------------------------|
| Aspect ratio     | `2 / 3`                              |
| Border radius    | `8px` (radius-md)                    |
| Hover scale      | `1.03`                               |
| Transition       | `transform 200ms ease, opacity 200ms ease` |
| Overlay color    | `rgba(0, 0, 0, 0.50)`               |
| Rating chip pos. | Bottom-right, `8px` inset            |
| Min width        | `120px` (thumbnail), `200px` (grid) |

### Usage Rules

**Use when:** displaying a film in any grid, list, or tier-row context where portrait imagery is available.

**Do not use when:** the layout demands a wide-format preview (use the Horizontal variant). Do not stretch the container to a non-2:3 ratio; always constrain via aspect-ratio utility. Do not show the rating chip in the resting state — it must only appear inside the hover overlay.

## Movie Poster — Horizontal

### Purpose

The horizontal poster presents a film in landscape orientation. It is used for editorial placements where cinematic framing matters more than density — featured section rows and the Recently Watched strip.

### Anatomy

1. **Image frame** — 16:9 aspect ratio container, `border-radius: 8px`
2. **Poster image** — fills frame via `object-fit: cover`
3. **Hover overlay** — semi-transparent scrim
4. **Play icon** — centered SVG icon, surfaces on hover

### States

| State | Transform     | Overlay opacity | Play icon |
|-------|---------------|-----------------|-----------|
| Rest  | `scale(1.00)` | `0`             | Hidden    |
| Hover | `scale(1.02)` | `1` (black/50)  | Visible   |

### Specs

| Property      | Value                                     |
|---------------|-------------------------------------------|
| Aspect ratio  | `16 / 9`                                  |
| Border radius | `8px` (radius-md)                         |
| Hover scale   | `1.02` (more subtle than vertical)        |
| Overlay color | `rgba(0, 0, 0, 0.50)`                    |
| Play icon     | `48px`, centered, `white`, opacity `0.9` |
| Transition    | `transform 200ms ease, opacity 200ms ease`|

### Usage Rules

**Use when:** the layout has horizontal room and cinematic context is appropriate — featured rows, hero-adjacent carousels, recently watched strips.

**Do not use when:** displaying dense grids or tier list thumbnails (use the Vertical variant). Do not use a play icon that implies immediate streaming if the action is navigation rather than inline playback — align the icon semantics with the actual behavior.

## Navbar

### Purpose

The Navbar provides persistent top-level navigation, brand identity, and primary actions. It is sticky to the top of the viewport and present on every page.

### Anatomy

1. **Root bar** — full-width, `height: 64px`, `background: black-600` (bg-surface)
2. **Logo** — left-aligned, links to `/`
3. **Nav links** — centered; `EXPLORE` and `COLLECTIONS` as plain text links
4. **Action group** — right-aligned; Search icon · PYF Button · Bell icon · Profile avatar

### States — Nav Links

| State   | Text color  | Token          |
|---------|-------------|----------------|
| Default | `white-700` | `text-white-700`|
| Hover   | `white-50`  | `text-white-50` |
| Active  | `white-50`  | `text-white-50` |

### Specs

| Property        | Value                         |
|-----------------|-------------------------------|
| Height          | `64px`                        |
| Background      | `black-600` / `bg-surface`    |
| Position        | `sticky`, `top: 0`, `z-50`   |
| Logo height     | `32px`                        |
| Nav link size   | `14px`, `500` weight, `0.06em` tracking, all-caps |
| Nav link gap    | `32px` between items          |
| Action gap      | `16px` between icons/button   |
| Icon size       | `20px` (search, bell)         |
| Avatar size     | `32px`, `border-radius: 9999px`|

### Usage Rules

**Use when:** on all pages, without exception. The Navbar must not be hidden, replaced, or repositioned.

**Do not use when:** building isolated modal or full-screen overlays that require temporary nav suppression — in those cases, retain the Navbar beneath the overlay rather than removing it from the DOM.

**Common mistakes:** duplicating PYF Button elsewhere in the Navbar; reducing height below 64px on desktop; using `position: fixed` instead of `sticky` (breaks scroll context).

## Tier List

### Purpose

The Tier List allows users to rank movies across seven ordered tiers from S (best) down to F (worst). It is a structured, horizontally scrollable grid used on the Tier List feature page.

### Anatomy

1. **Tier row** — full-width horizontal strip per tier
2. **Tier label** — `40px` wide column, left-aligned; bold uppercase letter; colored background per tier
3. **Poster strip** — horizontally scrollable row of vertical Movie Poster thumbnails (smaller size)

<TierList />

### Tier Colors

| Tier | Label bg color | Semantic meaning |
|------|---------------|-----------------|
| S    | `purple-400`  | Exceptional      |
| A    | `purple-300`  | Excellent        |
| B    | `purple-200` (tint) | Good       |
| C    | neutral       | Average          |
| D    | `red-300` (tint) | Below average |
| E    | `red-400` (tint) | Poor          |
| F    | `red-500`     | Unwatchable      |

### Specs

| Property          | Value                              |
|-------------------|------------------------------------|
| Label column width| `40px`                             |
| Label background  | Per tier table above               |
| Label text        | `white`, bold (`700`), uppercase   |
| Label font size   | `18px`                             |
| Row height        | `88px` (accommodates thumbnail)    |
| Thumbnail size    | `~56px` wide (2:3 ratio, ~84px tall — clipped to row) |
| Thumbnail gap     | `8px`                              |
| Poster strip      | `overflow-x: auto`, `scroll-snap-type: x mandatory` |
| Background        | `black-900`                        |

### Usage Rules

**Use when:** displaying a user's ranked film list in the dedicated Tier List view.

**Do not use when:** a simple ordered list or grid is sufficient. Do not reorder the tier sequence (S → F is fixed). Do not change the tier label letters — they carry established meaning for the target audience.

## Movie Card (Grid)

### Purpose

The Movie Card is the standard unit for browsing contexts — Explore page grids, search results, and collection views. It combines the vertical poster with metadata below it.

### Anatomy

1. **Poster** — Vertical Movie Poster component (2:3, 8px radius, with hover behavior)
2. **Title** — film title, below the poster
3. **Rating chip** — small badge showing score or star rating, positioned below the title or overlaid on the poster on hover

### Specs

| Property       | Value                                  |
|----------------|----------------------------------------|
| Poster         | Full-width of card column, 2:3 ratio   |
| Card width     | Column-driven (grid sets width)        |
| Title size     | `13px`, `500` weight                   |
| Title color    | `white-50`                             |
| Title truncation | Single line, `text-overflow: ellipsis`|
| Rating chip    | `12px` text, `black-600/80` bg, `4px` radius |
| Gap (poster → title) | `8px`                          |
| Grid gap       | `16px` column, `24px` row             |

### Usage Rules

**Use when:** rendering browsable film inventories in Explore, search results, or any collection view.

**Do not use when:** a single featured film needs prominent display (use the Horizontal Poster in a featured row instead). Do not omit the title — the rating chip alone is insufficient for accessibility. Ensure the title always renders even when the poster image fails to load.

**Common mistakes:** applying custom aspect ratios to the poster within a card; showing the rating chip in the resting state without the hover overlay (it should follow the Vertical Poster hover rules unless placed statically below the title).
